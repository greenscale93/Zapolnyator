# Zapolnyator — автоматическое заполнение отчёта ДКП из выгрузки 1С

**Zapolnyator** — это система, которая принимает через Telegram два Excel-файла (шаблон отчёта и выгрузку данных) и автоматически заполняет листы отчёта по заданным правилам. Агент не использует LLM — всё работает по жёстким правилам, описанным в JSON-конфиге. Подходит для ежемесячного формирования отчёта департамента корпоративных продаж.

## Особенности

- Принимает Excel-файлы через Telegram-бота.
- Заполняет листы «Реализация», «ДС», «ФОТ» (с группировкой по сотрудникам и разделением окладов/премий).
- Обрабатывает лист «Взаиморасчеты» со схлопыванием дублей, разделением по БДР/БДДС и маппингом офисов.
- Фильтрует данные по сценарию «ФАКТ», исключает подразделение «ГКП 10.6 (Емельянова)» и ВГО-строки.
- Автоматически определяет месяц и год из имени файла выгрузки (например, `ВыгрузкаДляExcel_2026-06`).
- Поддерживает группировку и агрегацию (суммирование, объединение комментариев).
- Очищает числовые значения от неразрывных пробелов (`\xa0`) и других мусорных символов.
- Работает без DeepSeek и других LLM — только правила и код.
- Имеет режим автотеста (`/autotest`), который запоминает последние загруженные файлы и автоматически запускает обработку при `/start`.
- Все сервисы упакованы в Docker и могут быть развёрнуты на VPS одной командой.

## Архитектура

Система состоит из трёх микросервисов, каждый в отдельном контейнере:

| Сервис | Назначение | Технологии |
|---|---|---|
| **Telegram Gateway** | Приём файлов, управление диалогом, отправка сообщений и готового файла | aiogram, aiohttp, Redis |
| **Orchestrator** | Координация процесса: чтение правил, вызов Worker, управление сессиями, отправка уведомлений | FastAPI, Redis, httpx |
| **Worker** | Выполнение основной работы: чтение Excel, фильтрация, группировка, запись в Excel | FastAPI, openpyxl, pandas, msoffcrypto-tool |
Связь между сервисами — по REST API через внутреннюю сеть Docker. Состояние задач хранится в Redis.

Требования
Docker и Docker Compose (для развёртывания)

Python 3.11+ (для локальной разработки)

Telegram Bot Token (получить у @BotFather)

(Опционально) учётная запись Docker Hub для избежания лимитов

Установка и запуск
Клонирование репозитория
bash
git clone git@github.com:greenscale93/Zapolnyator.git
cd Zapolnyator
Переменные окружения
Создайте файл .env (или задайте переменные через docker-compose.yml):

env
TELEGRAM_BOT_TOKEN=ваш_токен_бота
TEMP_DIR=/app/temp
ORCHESTRATOR_URL=http://orchestrator:8000
WORKER_URL=http://worker:8000
REDIS_URL=redis://redis:6379
AUTO_TEST=true   # опционально
AUTO_START_CHAT_ID=123456789   # опционально, chat_id для автозапуска
Запуск через Docker Compose
bash
docker compose up -d --build
Сервисы будут доступны:

Telegram Gateway — порт 8000 (для внешних вызовов, не используется напрямую)

Orchestrator — порт 8001 (внутренний)

Worker — порт 8002 (внутренний)

Redis — порт 6379 (внутренний)

Локальная разработка
Если вы хотите запускать сервисы вне Docker:

Установите зависимости для каждого сервиса (в папках telegram-gateway, orchestrator, worker):

bash
pip install -r requirements.txt
Запустите Redis локально (или укажите другой URL).

Запустите каждый сервис отдельно:

bash
# в папке telegram-gateway
python -m src.main
# в папке orchestrator
uvicorn src.main:app --host 0.0.0.0 --port 8001
# в папке worker
uvicorn src.main:app --host 0.0.0.0 --port 8002
Использование
Команды бота
/start — начать работу (при включённом автотесте сразу запускает обработку последних файлов).

/reset — сбросить состояние.

/autotest — включить/выключить режим автотеста (сохраняется индивидуально для пользователя).

/stop — принудительно остановить текущую задачу.

Загрузка файлов
Отправьте боту шаблон отчёта (Excel-файл, например, ДКП 10 - Май 2026.xlsx).

Отправьте выгрузку данных (Excel-файл с именем, содержащим ВыгрузкаДляExcel, например, ВыгрузкаДляExcel_2026-06.xlsx).

Месяц и год определяются автоматически из имени файла (год — 4 цифры, месяц — 2 цифры после дефиса или подчёркивания).

Бот применит правила из rules.json и вернёт заполненный файл.

Порядок отправки не важен. Можно отправить файлы одновременно.

Настройка правил заполнения (rules.json)
Все правила хранятся в файле orchestrator/rules.json. Он имеет следующую структуру:

json
{
  "sheets": {
    "ИмяЛиста": {
      "source_columns": {
        "номер_колонки_в_шаблоне": "имя_колонки_в_источнике"
      },
      "append_to_end": true,
      "header_row": 2,
      "filters": {
        "колонка_источника": "значение"
      },
      "exclude_filters": {
        "колонка_источника": "значение_для_исключения"
      },
      "view_filters": {
        "колонка_источника": ["допустимое_значение1", "допустимое_значение2"]
      },
      "group_by": ["колонка1", "колонка2"],
      "aggregations": {
        "колонка_для_агрегации": "sum | concat | concat_premia"
      }
    }
  }
}
Поддерживаемые агрегации
sum — суммирование числовых значений.

concat — объединение строк через "; ".

concat_premia — объединение комментариев только из строк с видом начисления «Премия» (используется для листа ФОТ).

Пример для листа «Реализация»
json
"Реализация": {
  "source_columns": {
    "3": "Подразделение",
    "4": "Сценарий",
    "5": "{month} {year}",
    "6": "Контрагент",
    "7": "Проект",
    "8": "СуммаСНДС",
    "9": "СуммаБезНДС",
    "10": "СтавкаНДС",
    "11": "Этап",
    "12": "НомерК7"
  },
  "append_to_end": true,
  "header_row": 2,
  "filters": {
    "Сценарий": "ФАКТ",
    "ТипЗаписи": "Реализация"
  },
  "exclude_filters": {
    "Подразделение": "ГКП 10.6 (Емельянова)"
  }
}
Разработка и доработка
Как добавить новый лист
Добавьте новый объект в sheets в rules.json.

Укажите:

source_columns — соответствие колонок шаблона и источника.

filters и exclude_filters при необходимости.

Если нужна группировка — укажите group_by и aggregations.

Перезапустите контейнеры (или просто замените файл и перезапустите Orchestrator).

Как изменить логику обработки данных
Основная логика обработки распределена по модулям Worker-сервиса:

- **`src/business_logic/process_data.py`** — оркестрация полного цикла обработки `process_data()`.
- **`src/business_logic/column_mapper.py`** — определение и разрешение колонок (авто/пользовательский маппинг).
- **`src/business_logic/filtering.py`** — фильтрация строк по сценарию, подразделению, ВГО.
- **`src/business_logic/payroll_processor.py`** — группировка зарплаты по сотрудникам.
- **`src/business_logic/settlement_processor.py`** — трёхветочная логика взаиморасчетов (внутренние/другие офисы/обычные).
- **`src/business_logic/ffot_processor.py`** — расчёт ФактическийФОТ с исключениями.
- **`src/excel_processor.py`** (`apply_sheet_mapping`) — низкоуровневая работа с Excel: фильтрация, группировка, вставка данных в шаблон.
- **`src/vz_processing.py`** — специфическая предобработка взаиморасчетов перед вставкой.

Для добавления нового типа обработки достаточно создать новый модуль в `worker/src/` и подключить его в нужном месте цепочки.

Планы по развитию

- Реализовать вычисление итоговых показателей (метрики) и запись их в отчёт.
- Поддержка большего числа типов выгрузок (например, CSV).
- Веб-интерфейс для настройки правил.

## Ключевые файлы

### Конфигурация
- **`rules.json`** — основной конфигурационный файл. Определяет, какие листы заполнять, какие колонки куда копировать, какие фильтры применять и как группировать данные. Изменение этого файла не требует пересборки — достаточно перезапустить Orchestrator.

### Orchestrator
- **`src/agent.py`** — ядро координации. Читает `rules.json`, последовательно вызывает Worker для каждого листа, передавая текущий путь к выходному файлу. Отправляет уведомления в Telegram через `TelegramNotifier`.
- **`src/agent/mapping_handler.py`** — управление маппингом офисов для взаиморасчетов: построение inline-клавиатур, обработка ответов, CRUD маппингов.
- **`src/telegram_notifier.py`** — отправка сообщений и документов в Telegram конкретному пользователю или в глобальный чат.
- **`src/session_manager.py`** — хранение состояний задач и настроек автотеста в Redis.
- **`src/memory.py`** — хранение правил и исключений (подразделения, сотрудники, НДС) в SQLite.
- **`src/api_models.py`** — Pydantic-модели запросов REST API.

### Worker
- **`src/excel_processor.py`** — выполняет всю работу с Excel: чтение данных, фильтрацию, группировку с агрегацией, очистку чисел, запись в целевой файл. Реализована логика `concat_premia` и очистка от неразрывных пробелов.
- **`src/vz_processing.py`** — предобработка взаиморасчетов: вычисление направления, разделение по БДР/БДДС, заполнение пустых маппингов, схлопывание дублей.
- **`src/business_logic/process_data.py`** — оркестратор полного цикла обработки данных: определение колонок → фильтрация → группировка → payroll → взаиморасчеты → ФОТ.
- **`src/vz_utils.py`** — поиск контрагентов с пустым маппингом офиса.
- **`src/template_reader.py`** — чтение списка офисов из листа «Отчетность БИТ 2026» шаблона.

### Telegram Gateway
- **`src/handlers/`** — пакет обработчиков, разбитый на модули:
  - `commands.py` — команды `/start`, `/reset`, `/autotest`, `/stop`, `/edit_mapping`
  - `documents.py` — приём и сохранение Excel/MXL-файлов
  - `polling.py` — фоновый мониторинг статуса задачи
  - `callbacks.py` — обработка inline-кнопок (выбор офиса, удаление маппинга)
  - `fsm.py` — состояния ожидания ответа пользователя
- **`src/utils/file_storage.py`** — сохранение/загрузка последних файлов для автотеста, извлечение месяца и года из имени файла.
